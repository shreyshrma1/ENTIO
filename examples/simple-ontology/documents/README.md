# Entio Document-Ingestion Test Templates

Templates:

- `001-business-policy-document-template.md`
- `002-grc-compliance-document-template.md`

Completed fictional examples:

- `commercial-account-and-payment-authorization-policy.pdf`
- `consumer-lending-servicing-compliance-standard.pdf`

The examples are written as controlled Meridian Community Bank documents. They use fictional people, organizations, accounts, loans, payments, and control records. Their subject matter is intentionally adjacent to the `simple-ontology` financial model without copying its named records or presenting ontology definitions.

Both templates contain structured tables and unstructured narrative sections.

## Suggested Codex Instruction

Use the selected template to create a realistic test document that fits the current Entio model ontology.

Requirements:

- Preserve the headings and overall structure.
- Replace every placeholder with realistic content.
- Keep the document's subject matter reasonably related to the current ontology; direct reuse of its classes, properties, or named individuals is optional.
- Include a small number of reasonable new concepts.
- Include explicit and implied relationships.
- Include dates, amounts, identifiers, roles, and statuses.
- Include one ambiguity, one conflict, and one later statement that extends or revises an earlier meaning.
- Do not include raw RDF, Turtle, JSON, or implementation instructions.
- Produce a business-readable document rather than a test script.
